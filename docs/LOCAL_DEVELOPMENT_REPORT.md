# Local Development Setup Report

## Issue

The backend startup command did not load `backend/.env`, and the existing file used the obsolete `JWT_SECRET` variable. Required JWT access/refresh and usable development SMTP values were therefore absent.

## Changes

- Created `backend/.env` with generated local-only JWT secrets, MongoDB, CORS, SMTP, and Super Admin settings.
- Created `backend/.env.example` without usable secrets.
- Updated backend development, migration, and seed scripts to load `.env` through Node.js `--env-file`.
- Added `SMTP_AUTH` and `SMTP_SECURE` configuration support.
- Added `npm run smtp:dev`, backed by the development-only `smtp-server` package.
- Disabled STARTTLS in the localhost SMTP sink to avoid autogenerated certificate failures.
- Removed a duplicate refresh-session expiry index declaration.
- Fixed the admin Vite development and preview port at `127.0.0.1:5174`.

## Local Commands

From `backend/`, run the SMTP sink and backend in separate terminals:

```sh
npm run smtp:dev
npm run dev
```

From `admin/`:

```sh
npm run dev
```

The admin panel is available at `http://127.0.0.1:5174`.

## Dependencies

MongoDB (`mongod` and `mongosh`) was already installed and running. Node.js and npm were already installed. The only missing development dependency was the local SMTP sink; `smtp-server` and `@types/smtp-server` were added to backend development dependencies.
